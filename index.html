Features

Everything automatic.
Nothing in the way.

Zero-config defaults that work from install. On-demand commands when you want more control. Nothing in between.

01

Tool output compression

File reads become skeletons — imports, signatures, types. Bash returns errors and summary. Grep caps at 25 matches. Automatic, always on.

02

Live token meter

Exact context window usage in the status line — mode-aware, always visible. Know where you stand before you hit the wall.

03

Auto-compact at 70%

History is summarized automatically before the window fills. Sessions run indefinitely. No more "context limit reached" mid-task.

04

Project wiki

First session in a new project, Pith builds a wiki. Decisions, architecture, entities — it writes itself as you work. Searchable anytime.

05

Output modes

Lean for everyday work. Precise for docs and PRs. Ultra when every token counts. Switch with a single command.

06

Token budgets

Hard ceilings that Pith enforces — Claude won't exceed the limit without warning you first. Full cost control, no babysitting.

Commands

Type it.
Done.

Every command works inside any Claude Code session — no flags, no setup, no config files.

/pith leanDrop articles, short synonyms, fragments OK — tightest default
/pith preciseProfessional and tight — no filler, full sentences kept
/pith ultraMaximum compression — arrows →, tables over prose, abbreviate freely
/pith offBack to normal Claude output, no compression
/pith debugStructured debug: hypothesis → test → result
/pith reviewCode review: issues → suggestions → verdict
/pith archArchitecture: context → decision → trade-offs
/pith wiki "<question>"Search the project wiki
/pith ingest <file>Add a source doc or spec to the wiki
/pith statusToken usage breakdown for this session
/budget <N>Hard token ceiling — enforced, not suggested
/pith commitCommit format: summary → changes → why
